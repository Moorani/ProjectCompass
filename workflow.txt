WORKFLOW 1: Browse & Explore Project Ideas
[Branch: workflow-1]
----------------------------------------------------------------

Description:
This workflow covers how a student user browses and explores
available software project ideas through the platform without
any prior filtering or preferences.

Step-by-Step:

1. User lands on the ProjectCompass homepage.
2. The homepage displays a brief problem statement explaining
   why choosing a project is difficult for students.
3. User reads the short "How it Works" section on the homepage.
4. User clicks the "Explore Project Ideas" call-to-action button.
5. The system navigates the user to the Project Categories page.
6. User sees all 6 categories listed:
   - Web Applications
   - Mobile Applications
   - AI / ML Projects
   - Systems & Tools
   - Data-Driven Projects
   - Social Impact Projects
7. User optionally uses the search bar to type a keyword
   (e.g. "budget", "React", "beginner").
8. User optionally applies filters:
   - Filter by Difficulty (Beginner / Intermediate / Advanced)
   - Filter by Category
   - Filter by Tech Stack
9. The page dynamically updates to show matching project cards.
10. User clicks on a project card of interest.
11. The system navigates to the Project Detail Page for that idea.
12. User reads the full brief:
    - Problem Statement
    - Target Users
    - Core Features (minimum scope)
    - What NOT to Build
    - Optional Extensions
    - Suggested Tech Stack
    - Estimated Effort
13. If the project suits them, the user notes it down or
    begins planning.
14. User can click "Back to Projects" to continue browsing.

Key Features Used:
- Homepage with CTA
- Project Categories Page
- Search and Filter System
- Project Cards
- Project Detail Page

WORKFLOW 2: Narrow My Project (Guided Decision Flow)
[Branch: workflow-2]
----------------------------------------------------------------

Description:
This workflow covers the guided decision-making flow where a
student answers a short questionnaire and the system uses
rule-based logic to recommend the most suitable project ideas.

Step-by-Step:

1. User lands on the ProjectCompass homepage.
2. User clicks the "Narrow My Project" call-to-action button.
   (Alternatively, user navigates to it from the top navbar.)
3. The system opens the Guided Flow page with a 5-step form.
4. A progress bar at the top shows current step out of 5.

   Step 1 - Skill Level:
   User selects one of:
   - Beginner (learning fundamentals, few projects done)
   - Intermediate (comfortable with basics, a few projects done)
   - Advanced (strong foundation, built production-like apps)

   Step 2 - Area of Interest:
   User selects one category they are most interested in:
   - Web Apps / Mobile Apps / AI & ML /
     Systems & Tools / Data Projects / Social Impact

   Step 3 - Time Availability:
   User selects their weekly time commitment:
   - Low (less than 6 hrs/week)
   - Medium (6 to 12 hrs/week)
   - High (more than 12 hrs/week)

   Step 4 - Solo or Team:
   User selects:
   - Solo (building alone)
   - Team (2 to 4 members)

   Step 5 - Primary Goal:
   User selects:
   - Learning (deepen understanding of a technology)
   - Portfolio (showcase work to future employers)

5. After the final selection, the system applies deterministic
   rule-based filtering logic:
   - Hard rejects projects beyond the user's skill level
   - Removes projects that exceed time availability
   - Scores remaining projects by category match and difficulty
     alignment
   - Sorts results by match score (highest first)

6. The system displays up to 6 recommended project cards.
7. If a scope risk is detected (e.g. large project + low time),
   a warning banner is shown at the top of results.
8. User browses the recommended projects.
9. User clicks a project card to view its full detail page.
10. If unsatisfied, user clicks "Start Over" to redo the flow.

Key Features Used:
- Guided 5-step questionnaire
- Rule-based filtering (no AI)
- Scope warning system
- Results page with filtered project cards
- Project Detail Page

WORKFLOW 3: Admin - Manage Project Ideas
[Branch: workflow-3]
----------------------------------------------------------------

Description:
This workflow covers how an admin user logs into the admin
panel and manages the project idea database by adding, editing,
or deleting entries.

Step-by-Step:

1. Admin clicks the "Admin" badge in the top navigation bar.
2. The system navigates to the Admin Login page.
3. Admin enters their username and password credentials.
4. On successful login, the system navigates to the Admin
   Dashboard.
5. The dashboard sidebar shows two sections:
   - Overview (stats and category breakdown)
   - Projects (manage all project entries)

   --- Overview Tab ---
6. Admin views summary statistics:
   - Total number of projects
   - Number of categories
   - Count of Beginner / Intermediate / Advanced projects
7. Admin views a visual bar chart showing projects per category.

   --- Projects Tab ---
8. Admin clicks "Projects" in the sidebar.
9. A table lists all existing projects with columns:
   - Title, Category, Difficulty, Estimated Effort, Actions
10. To ADD a project:
    a. Admin clicks the "+ Add Project" button.
    b. A modal form opens.
    c. Admin fills in all required fields:
       - Title
       - Category (dropdown)
       - Difficulty (dropdown)
       - Estimated Effort
       - Tech Stack (comma-separated)
       - Short Description
       - Problem Statement
       - Target Users
       - Core Features (one per line)
       - What NOT to Build (one per line)
       - Optional Extensions (one per line)
    d. Admin clicks "Add Project".
    e. The new project appears in the table and on the site.

11. To EDIT a project:
    a. Admin clicks the "Edit" button next to any project row.
    b. The same modal opens, pre-filled with existing data.
    c. Admin modifies desired fields and clicks "Save Changes".
    d. Changes are reflected immediately across the platform.

12. To DELETE a project:
    a. Admin clicks the "Delete" button next to a project row.
    b. The project is removed from the table and the platform.

13. Admin clicks "Log out" in the sidebar to end the session.

Key Features Used:
- Admin Login with credential validation
- Admin Dashboard with stats overview
- Project management table (view all)
- Add Project modal form
- Edit Project modal form
- Delete Project functionality
- Session logout
